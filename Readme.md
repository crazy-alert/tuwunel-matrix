# 🚀 Matrix-сервер на Tuwunel с Docker Compose

Этот репозиторий содержит готовый Docker‑стек для развёртывания собственного Matrix‑сервера на базе **Tuwunel** (лёгкий сервер на Rust), PostgreSQL, Caddy (автоматический HTTPS) и Unbound (кеширующий DNS). Всё это упаковано в удобный `docker-compose.yml` – достаточно склонировать, поправить конфиги и запустить.

---

## 📁 Структура репозитория
```text
├── .env # основной конфигурационный файл
├── caddy-config/Caddyfile # конфигурация обратного прокси (Caddy), генерируется с учётом .env
├── docker-compose.yml # описание всех сервисов
├── unbound-conf/ # каталог для дополнительных конфигов Unbound
│ └── 99-local.conf #  настройки кеширования и форвардеров
├── unbound-data/ # данные Unbound (кеш, ключи DNSSEC) – будет создан автоматически
└── README.md # этот файл
```
---
## 🔧 Предварительные требования

- Сервер (VPS или выделенный) с **Ubuntu 20.04+ / Debian 11+**, минимум 1 GB RAM (рекомендуется 2 GB).
- Установленные **Docker** и **Docker Compose** (обычно `docker compose` входит в состав Docker).
- Доменное имя, направленное на IP вашего сервера. Понадобятся два поддомена:
    - `matrix.ваш-домен.ru` – для сервера Matrix
    - (опционально) `element.ваш-домен.ru` – если позже захотите поставить веб‑клиент Element
- Открытые порты в фаерволе: **80**, **443**, **8448** (TCP).

## ⚙️ Настройка перед запуском

### 1. Клонируйте репозиторий и перейдите в него

Создать директорию в которую установим, например `/opt/tuwunel-matrix`, перейти в неё и скопировать этот репозиторий в неё
```bash
mkdir /opt/tuwunel-matrix
cd /opt/tuwunel-matrix
git clone -v  https://github.com/crazy-alert/tuwunel-matrix.git .
```

Скопируйте файл примеры конфигурации и отредактируйте его:
```bash
cp .env.example .env
nano .env
```
Это главный конфиг сервера. Обязательно замените:
server_name – ваш домен (например, `matrix.example.ru`).

⚠️ Не забудьте после создания первого администратора отключить регистрацию – установить `TUWUNEL_REGISTRATION_ENABLED=false`.
---
### 🚀 Запуск сервера
Выполните в каталоге с ```docker-compose.yml```:

```bash
docker-compose up -d
```
Если появилась ошибка
`Command 'docker-compose' not found, but can be installed with:
apt install docker-compose` - значит не установлен docker, устанавливаем командой:
```bash
apt install docker compose
```
После чего снова пытаемся запустить:
```bash
docker compose up -d
```

Через минуту все контейнеры будут запущены. Проверьте логи:
```bash
docker compose logs -f
```
Если всё хорошо, вы увидите, что Caddy получил сертификаты, а Tuwunel подключился к БД.

### 👤 Создание первого администратора
1. Установите на телефон Element X (iOS / Android) или любой другой Matrix‑клиент.
2. При входе укажите ваш домашний сервер: ```https://matrix.ваш-домен.ru```
3. Зарегистрируйте нового пользователя:
   * Логин и пароль – любые
   * Когда клиент перенаправит вас в браузер, введите токен регистрации, который вы указали в `.env` ```TUWUNEL_REGISTRATION_TOKEN=msecret123```.
4. После успешной регистрации этот пользователь автоматически станет глобальным администратором.

Теперь можно отключить открытую регистрацию:
* Остановите стек: ```docker compose down```
* В .env измените `TUWUNEL_REGISTRATION_ENABLED=true` на `TUWUNEL_REGISTRATION_ENABLED=false`.
* Запустите снова: ```docker compose up -d```

Регистрация новых пользователей теперь возможна только через пригласительные токены, которые вы можете создавать прямо из Matrix‑комнаты администратора (#admins). Например, команда:

```text
!admin token create --usage-limit 1 для_друга
```

Создаст одноразовый токен, который можно отправить другу в любом мессенджере.

### 🌐 Порты и безопасность
В данной конфигурации наружу открыты только порты 80, 443 и 8448 – все они обслуживаются Caddy. Остальные сервисы (PostgreSQL, Tuwunel, Unbound) доступны только внутри Docker‑сети ```internal``` и не видны из интернета.
Проверить открытые порты можно командой:
```bash
sudo ss -tulpn | grep LISTEN
```
Должны быть видны только :80, :443 и :8448.

### 📦 Управление стеком
* Остановить: ```docker compose down```
* Запустить: ```docker compose up -d```
* Перезапустить конкретный сервис: ```docker compose restart tuwunel```
* Посмотреть логи: ```docker compose logs -f [имя_сервиса]```
* Обновить образы: ```docker compose pull && docker compose up -d```

### ❓ Часто задаваемые вопросы
####❔ Почему Unbound не работает / не резолвит?
Убедитесь, что в ```docker-compose.yml``` у сервисов ```tuwunel``` и ```caddy``` прописана опция ``` dns: 172.20.0.53```. Проверьте, что Unbound запустился:

```bash
docker compose logs unbound
```
Если нужна отладка, зайдите внутрь контейнера Unbound:

```bash
docker exec -it tuwunel-unbound sh
drill matrix.org @127.0.0.1
```
#### ❔ Как добавить ещё одного администратора?
Первый администратор может выдавать права другим через команду в комнате #admins:
```text
!admin make @пользователь:ваш-домен.ru admin
```
#### ❔ Хочу поставить веб‑клиент Element.
Добавьте в ```docker-compose.yml``` сервис ```element``` и настройте ещё один поддомен в ```Caddy```. Это выходит за рамки данного README, но в интернете много готовых примеров.

#### ❔ Как сделать бэкап?
Остановите стек и скопируйте каталоги ```postgres-data/```, ```tuwunel-data/``` и ```unbound-data/``` в надёжное место.
